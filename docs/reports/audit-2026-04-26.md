### Specific risk vectors checked

| Vector | Files | Verdict |
|---|---|---|
| API key leakage in source | All `*/client.ts` | All keys are `process.env.*` only — no logging, no client-side exposure |
| Public API routes | `app/api/*` | Input validation on `?topic=` and POST body; runtime: nodejs |
| URL-param injection (XSS) | `app/teleprompter/page.tsx`, `ContentDetailView.tsx` | React's auto-escaping handles all rendering; no `dangerouslySetInnerHTML` |
| `mailto:` header injection | `GapCard.tsx`, `PitchCard.tsx` | All values pass through `encodeURIComponent` — safe |
| External URL content → LLM (prompt injection) | `lib/pipeline/remixes.ts` | Content stays inside Gemini's structured-output sandbox; output schema enforced |
| Filesystem writes from API | `app/api/generate/route.ts` | Writes confined to `data/slate/` with stamped filenames; slug derived from project name with regex sanitization |
| `localStorage` usage | `app/teleprompter/page.tsx`, `ContentDetailView.tsx` | Stores user preferences only; no credentials |
| `navigator.clipboard.writeText` | `ContentDetailView.tsx` | Writes generated script (LLM output) — no credential exposure |
| Placeholder integrity in `.env.example` | `.env.example` | `skc-...`, `AIza...`, `tvly-dev-...` placeholders only |

### History note

Earlier in this repo's history, commit `89a2dd6` accidentally embedded real Peec + Gemini keys in `.env.example`. Google's secret-leak scanner detected the Gemini key within hours and revoked it. The Peec key was rotated immediately. Both keys are now invalid. Commit [`c556ac3`](https://github.com/soheilfathalian/VAIRAL/commit/c556ac3) scrubbed the placeholders.

This audit confirms no real keys remain in HEAD or any newer commits.
